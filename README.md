- src/ — исходные файлы проекта

## Данные и типы данных, используемые в приложении

Карточка товара
```
interface IProductItem {
    title: string; // - Название товара
    description: string // - Описание товара
    image: string; // - Ссылка на изображение товара
    category: string // - Категория товара
    price: number //- Цена товара
}
```
Заказ
```
interface IOrder {
    customer: ICustomer; // информация о покупателе
    items: IProductItem[]; // список товаров в заказе
}
```
Покупатель
```
interface ICustomer {
    email: string; // - электронная почта покупателя
    phone: string; // - телефон покупателя
    address: string; // - адрес доставки покупателя
    payment:string; // - способ  оплаты
}
```
Данные для карточки товара
```
type TProductItemInfo = Pick<IProductItem,'title' | 'description' | 'image' | 'category' | 'price'>;
```
Данные для создания заказа
```
type TOrderMain = Pick<IOrder, 'items'>;
```
Тип объекта для запроса к серверу 
```
interface IApi {
    baseUrl: string
    get<T>(uri:string): Promise<T>
    post<T>(uri:string, data:object, method?: ApiPostMethods): Promise<T>
}
```
Данные покупателя для заказа
```
type TOrderCustomer = Pick<ICustomer, 'email' | 'phone' | 'payment' | 'address'>;
```
## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP: 
- слой представления, отвечает за отображение данных на странице, 
- слой данных, отвечает за хранение и изменение данных
- презентер, отвечает за связь представления и данных.

### Базовый код

#### Класс Api
Содержит в себе базовую логику отправки запросов. В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.
Методы: 
- `get` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер
- `post` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.

#### Класс EventEmitter
Брокер событий позволяет отправлять события и подписываться на события, происходящие в системе. Класс используется в презентере для обработки событий и в слоях приложения для генерации событий.  
Основные методы, реализуемые классом описаны интерфейсом `IEvents`:
- `on` - подписка на событие
- `emit` - инициализация события
- `trigger` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие

### Слой данных

#### Класс ProductItemModel
Класс отвечает за хранение и логику работы с данными карточек товаров.\
Конструктор класса принимает инстант брокера событий\
Данные полей класса:
- productItems: IProductItem[] - массив карточек товара, загруженный с сервера
- events: IEvents - экземпляр класса `EventEmitter` для инициации событий при изменении данных.\
Методы класса:
- getProductItem(): IProductItem - получить отдельную карточку
- сеттеры и геттеры для сохранения и получения данных из полей класса

#### Класс OrderModel
Класс отвечает за хранение данных о списке выбранных для заказа товаров и покупателе.\
Доступ к полям класса организован через сеттеры и геттеры
Конструктор класса принимает инстант брокера событий\
Данные полей класса:
- orderItems: iProductItem[] - массив выбранных товаров отдельного заказа
- customer: ICustomer - данные покупателя для формирования заказа
- events: IEvents - экземпляр класса `EventEmitter` для инициации событий при изменении данных.

Методы класса:
   - getProductItem(productItemId: string): IProductItem - возвращает карточку товара по id
   - addProductItem(productItem:IProductItem): void - добавляет товар в заказ и вызывает событие изменения массива карточек товаров в заказе
   - removeProductItem(productItemId:string, payload: Function | null): void - удаляет товар из заказа и вызывает событие изменения массива карточек товаров в заказе
   - сеттеры и геттеры для сохранения и получения данных из модели

### Классы представления
Все классы представления отвечают за отображение передаваемых в них данных внутри контейнера(DOM-элемент) .
#### Класс Popup
Отрисовывает модальное окно. Так же предоставляет методы `open` и `close` для управления отображением модального окна. Устанавливает слушатели на клавиатуру, для закрытия модального окна по Esc, на клик в оверлей и кнопку-крестик для закрытия попапа.\
\
constructor (selector: string, templateId: string events: IEvents).\
\
Конструктор принимает селектор, по которому в разметке страницы будет идентифицировано модальное окно, уникальный идентефикатор шаблона формы, по которому последовательно будут выбраны формы для отработки каждого этапа сценария покупателя и экземпляр класса `EventEmitter` для возможности инициации событий.\


Поля класса
- popup: HTMLElement - элемент модального окна
- templateId - уникальный идентефикатор шаблона формы
- events: IEvents - брокер событий

#### Класс PopupForm
Расширяет класс Popup. Предназначен для реализации модального окна с формой содержащей поля ввода.\
При сабмите инициирует событие, передавая в него объект с данными из полей ввода формы.\
При изменении данных в полях ввода инициирует событие изменения данных. \
Предоставляет методы для отображения ошибок и управления активностью кнопки сохранения.\
Поля класса:
- submitButton: HTMLButtonElement - Кнопка отправки формы
- form: HTMLFormElement - элемент формы
- formName: string - значение атрибута name формы
- templateId: string - значение атрибута Id у шаблона формы
- inputs: NodeListOf<HTMLInputElement> - коллекция всех полей ввода формы
- errors: Record<string, HTMLElement> - объект хранящий все элементы для вывода ошибок под полями формы с привязкой к атрибуту name инпутов

Методы:
- setValid(isValid: boolean): void - изменяет активность кнопки подтверждения
- getInputValues(): Record<string, string> - возвращает объект с данными из полей формы, где ключ - name инпута, значение - данные введенные пользователем
- setInputValues(data: Record<string, string>): void - принимает объект с данными для заполнения полей формы
- setError(data: { field: string, value: string, validInformation: string }): void - принимает объект с данными для отображения или сокрытия текстов ошибок под полями ввода
- showInputError (field: string, errorMessage: string): void - отображает полученный текст ошибки под указанным полем ввода
- hideInputError (field: string): void - очищает текст ошибки под указанным полем ввода
- close(): void - расширяет родительский метод дополнительно при закрытии очищая поля формы и деактивируя кнопку сохранения
- get form: HTMLElement - геттер для получения элемента формы


#### Класс ProductItem
Отвечает за отображение полного описания отдельного товара, реализует интерфейс IProductItem. 
Класс используется для отображения карточек товаров на главной странице сайта и в заказе. В конструктор класса передается DOM элемент темплейта, что позволяет при необходимости формировать карточки разных вариантов верстки. В классе устанавливаются слушатель кнопку добавления товара в заказ.\
\
Поля класса содержат элементы разметки элементов карточки.\
 Конструктор, кроме темплейта принимает экземпляр `EventEmitter` для инициации событий.\
 \
Методы:
- setData(productItemData: IProductItem): void - заполняет атрибуты элементов карточки товара данными.
- isInOrder(): boolean - метод возвращает состояние карточки товара (добавлена или нет в заказ)
- render(): HTMLElement - метод возвращает полностью заполненную карточку товара с установленными слушателями
- геттер id возвращает уникальный id карточки товара

#### Класс ProductItemsContainer
Отвечает за отображение блока с карточками товаров на главной странице. Предоставляет метод `addProductItem(ProductItemElement: HTMLElement)` для добавления карточек на страницу и сеттер `container` для полного обновления содержимого. В конструктор принимает контейнер, в котором размещаются карточки.

## Взаимодействие компонентов
Код, описывающий взаимодействие представления и данных между собой находится в файле `index.ts`, выполняющем роль презентера.\
Взаимодействие осуществляется за счет событий генерируемых с помощью брокера событий и обработчиков этих событий, описанных в `index.ts`\
В `index.ts` сначала создаются экземпляры всех необходимых классов, а затем настраивается обработка событий.

*Список всех событий, которые могут генерироваться в системе:*\
*События изменения данных (генерируются классами моделями данных)*
- `order:addNewProductItem` - установка статуса "выбрано" (inOrder = true)

*События, возникающие при взаимодействии пользователя с интерфейсом (генерируются классами, отвечающими за представление)*
- `productItem:open` - открытие модального окна с подробным описанием товара
- `order:open` - открытие модального окна со списком товаров для заказа (корзина)
- `order:removeItem` - удаление товара из заказа (корзины)
- `orderAddress:input` - изменение в поле адреса доставки в форме с выбором способа оплаты и вводом адреса доставки
- `orderEmail:input` - изменение данных в форме ввода email покупателя
- `orderTelephone:input` - изменение данных в форме ввода телефона покупателя
- `order:submit` - подтверждение данных, введенных в модальное окно
- `contacts:submit` - подтверждение данных, введенных в модальное окно
- `order:validation` - событие, сообщающее о необходимости валидации формы с адресом
- `customer:validation` - событие, сообщающее о необходимости валидации формы с контактами покупателя
